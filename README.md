# direct-mcp AI Project

Стартовый проект для управления рекламой в **Яндекс Директе** и **VK Ads** через ваш ИИ с помощью сервиса [lidfly.ru](https://lidfly.ru/).

Раскрывает мощь вашего ИИ (Клод, Курсор, ЧатГПТ и т.п.) при работе с рекламой:

1. Создает, анализирует, управляет рекламой в директе, как сениор маркетолог.
2. ВК АДС - забудьте про кабинет - делайте всё включая картинки в вашем ИИ, создавайте, понимайте эффективность, находите сотни групп для рекламы - все в чате с вашим ИИ.
3. Задачи - ИИ сам себе ставит задачи (проверь как сработало изменение бюджета через неделю и скорректируй по реальным данным) и сам их выполняет (тоже сразу из коробки)
4. Работа с вордстат и связь сразу с рекламой. "Посмотри почему просела реклама в пятницу, проанализируй в вордстат март предыдущего года, что было по пятницам?" Выдаст статистику и резюме причины.

Сервис для агентств и бизнесов - можно подключить и агентство и несколько отдельных кабинетов.

Клонируйте репозиторий, подставьте API-ключ — и управляйте кампаниями из любого AI-агента: Claude Code, ChatGPT, Cursor, Copilot, Windsurf, Gemini и других.

## Что такое direct-mcp

[direct-mcp](https://lidfly.ru/) — это сервис, который подключает AI-агентов к рекламным платформам. Работает как умный-слой между ИИ и API Яндекс Директа / VK Ads — вы общаетесь с агентом на естественном языке, а он управляет рекламой через API.

### Подключение — один эндпоинт (v3)

Все платформы подключаются через **один** эндпоинт — отдельные подключения для Директа, VK, Вордстата и Workspace больше не нужны:

| Эндпоинт                   | Что внутри                                                               |
| -------------------------- | ------------------------------------------------------------------------ |
| `https://lidfly.ru/mcp/v3` | Яндекс Директ + Метрика + Вордстат + VK Ads + Workspace (память и задачи) |

#### Как работает v3

Раньше AI-клиент получал сразу весь каталог — сотни инструментов, десятки KB в контексте. Это раздувало старт и упиралось в лимиты у ChatGPT и Claude Desktop.

v3 отдаёт короткий список из **6 meta-инструментов**: `search_tools` (поиск по каталогу), `get_tool_schema` (схема), `call_tool` (чтение), `call_write_tool` (запись/действие), `get_methodology` и `subscription_status`. Провайдерские инструменты (`get_campaigns`, `wordstat_top_requests`, …) не засоряют список — AI находит нужный через `search_tools`, подтягивает схему и вызывает по имени через `call_tool`/`call_write_tool`. За meta-слоем доступен весь каталог: ~119 инструментов Яндекс Директа, ~76 VK Ads, Вордстат и Workspace.

> **v3 в beta.** Эндпоинт уже стабильно работает и заметно удобнее: короткий контекст на старте, один ключ, одно подключение для всех платформ и любых клиентов — **Claude Code, Cursor, VS Code, Windsurf, Gemini CLI, ChatGPT, Claude Desktop**.

### Поддерживаемые AI-клиенты

ChatGPT, Claude Desktop, Claude Code, Cursor, VS Code (Copilot, Cline, Continue.dev), Windsurf, Gemini CLI, Cherry Studio, DeepSeek — любой клиент с поддержкой MCP.

## Быстрый старт

### 1. Клонируйте проект

```bash
git clone <url-репозитория>
cd direct-mcp-ai-project
```

### 2. Получите API-ключ

1. Зарегистрируйтесь на [lidfly.ru/app](https://lidfly.ru/app) — укажите email, подтвердите кодом
2. API-ключ сгенерируется автоматически
3. Добавьте OAuth-токен Яндекс Директа (кнопка "Получить токен на Яндексе" в кабинете — 30 секунд)
4. Для VK Ads — добавьте токен VK в том же кабинете

**7 дней бесплатного триала**, далее 990 руб/мес (для агентств — 4 990 руб/мес).

### 3. Настройте `LIDFLY_TOKEN`

Скопируйте шаблон токенов и впишите API-ключ в переменную `LIDFLY_TOKEN`:

```bash
cp tokens.env.example tokens.env
# затем заполните LIDFLY_TOKEN в tokens.env
```

Перед запуском агента загрузите переменные в текущую shell-сессию:

```bash
set -a
source tokens.env
set +a
```

Для Claude Code — файл `.mcp.json`:

```json
{
  "mcpServers": {
    "lidfly": {
      "type": "http",
      "url": "https://lidfly.ru/mcp/v3",
      "headers": {
        "Authorization": "Bearer ${LIDFLY_TOKEN}"
      }
    }
  }
}
```

Один сервер подключает сразу Яндекс Директ, Метрику, Вордстат, VK Ads и Workspace — отдельные подключения для каждой платформы больше не нужны.

Конфиги для Claude Code и Codex уже лежат в проекте — ключ должен быть в переменной окружения, а не в конфиге.

### 4. Запустите агента

```bash
# Claude Code
claude

# Gemini CLI
gemini

# Codex
codex
```

Или откройте папку проекта в Cursor / VS Code / Windsurf.

### 5. Попробуйте

```
покажи мои кампании
```

## Возможности

### Яндекс Директ (119 инструментов)

- **Кампании** — просмотр, создание, редактирование, остановка, возобновление, архивирование, изменение бюджетов
- **Группы объявлений** — структура кампаний, таргетинг, типы групп
- **Объявления** — заголовки, тексты, ссылки, быстрые ссылки, уточнения, модерация
- **Ключевые фразы** — управление фразами, ставками, статусами, минус-словами
- **Автотаргетинг** — настройка категорий запросов (целевые, узкие, широкие)
- **Статистика** — показы, клики, расход, CTR, цена клика, конверсии с группировкой по дням/неделям/месяцам
- **Поисковые запросы** — реальные запросы пользователей с расходом и конверсиями
- **Ретаргетинг** — аудитории и условия подбора

### Яндекс Wordstat (встроен)

- Частотность фраз (точные цифры запросов в месяц)
- Расширение семантики с похожими запросами
- Сезонность и тренды (динамика с 2018 года)
- Спрос по регионам с индексом интереса
- Автоматический подбор минус-слов

### VK Ads (76 инструментов)

- **Кампании** — создание, редактирование, массовое управление
- **Группы объявлений** — таргетинг, бюджеты, массовые операции
- **Объявления** — создание, модерация, шаблоны (11 инструментов)
- **Контент** — загрузка изображений
- **Статистика** — показы, клики, конверсии, CPA
- **Аудитории** — ремаркетинг, сегменты, списки ретаргетинга (20 инструментов)
- **Пакеты размещения** — форматы рекламы и таргетинги
- **Прогнозы** — оценка охвата при разных ставках
- **Контекстные фразы** — создание и управление
- **Лид-формы** — сбор лидов из объявлений
- **Справочники** — регионы, устройства, категории приложений
- **Агентства** — управление клиентами, трансферы средств

### Задачи и напоминания

ИИ может создавать задачи на проверку кампаний с email-напоминаниями — "проверь статистику через неделю", "напомни обновить объявления 1 марта".

### Скиллы (Claude Code)

Пять встроенных скиллов для управления, аналитики и оптимизации рекламных кампаний.

| Скилл               | Что делает                    | MCP-сервер      | Ключевые возможности                                                         |
| ------------------- | ----------------------------- | --------------- | ---------------------------------------------------------------------------- |
| **yandex-direct-campaign-builder** | Создание и управление кампаниями Директа | `lidfly` | Создание, оптимизация, масштабирование кампаний, ключевые слова, минус-слова |
| **vk-ads**          | Управление рекламой VK        | `lidfly`        | Кампании, таргетинг, ретаргетинг, контекстные фразы, look-alike              |
| **yandex-metrika**  | Аналитика через Метрику       | `lidfly` | UTM-отчёты с CPA, оценка здоровья кампании, сравнение периодов               |
| **demand-research** | Исследование спроса           | `lidfly` | Упущенный спрос, сезонный прогноз бюджета, каннибализация ключевых           |
| **serp-monitor**    | Мониторинг позиций            | —               | Позиции в Яндексе, анализ конкурентов, трекинг динамики                      |

**Настройка токенов:** скопируйте `tokens.env.example` в `tokens.env`, заполните `LIDFLY_TOKEN` для MCP и Yandex XML ключи для `serp-monitor`, если он нужен.

```bash
cp tokens.env.example tokens.env
# Заполните LIDFLY_TOKEN и, при необходимости, токены Yandex XML
```

## Для агентств

Тариф для агентств (4 990 руб/мес) включает:

- Неограниченное количество клиентов

- Кросс-клиентская аналитика — утренняя сводка по всем клиентам одной фразой
- Сравнение показателей между клиентами
- Подключение через OAuth-токен основного логина агентства

## Настройка под свой бизнес

После клонирования заполните два файла под свой проект:

1. **`PROJECTS.md`** — главный файл бизнес-логики:

   - Счётчик Яндекс Метрики и цели конверсии
   - Воронка конверсий (как срабатывает цель)
   - Пороги CPA (что считать хорошо/плохо)
   - Сезонность, гео-таргетинг, минус-слова
   - Именование кампаний

2. **`LEGAL.md`** — юридические ограничения для контента:

   - Запрещённые термины для вашей ниши
   - Допустимые формулировки
   - Актуально для медицины, финансов, образования и других регулируемых ниш

Память кампаний (клиенты, история изменений, решения, результаты по периодам, задачи) заполнять не нужно — она ведётся автоматически в **Workspace** (облако LidFly, часть единого эндпоинта `https://lidfly.ru/mcp/v3`) и общая для всех ИИ-клиентов. Агент сам пишет туда после каждой проверки и поднимает контекст через `workspace_get_context`. Отдельное подключение не нужно — Workspace работает через тот же v3-эндпоинт и API-ключ (см. `.mcp.json` / `.codex/config.toml`). Файлы `campaigns/*.md` оставлены только для разового импорта старых данных через `workspace_import_legacy_markdown`.

Агент будет учитывать все эти правила при анализе и создании кампаний.

## Безопасность

- OAuth-токен хранится на сервере в зашифрованном виде
- История запросов не сохраняется
- Сервер работает как прокси — ваш ИИ-агент не получает прямой доступ к API Яндекса/VK
- API-ключ проекта храните в `LIDFLY_TOKEN` / `tokens.env`; не записывайте живой ключ в `.mcp.json`, `.codex/config.toml` или другие файлы для git
