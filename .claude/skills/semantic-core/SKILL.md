# Semantic Core — Senior PPC and SEO Workflow

Сбор семантики не как джун ("Wordstat → залил → запустил"), а как сениор: от экономики и интента к структуре кампаний, страниц и контентных кластеров. Активируется при триггерных запросах из description.

## Главное правило

**Сначала ВОПРОСЫ → потом инструменты.** Без понимания экономики клиента семантика — это просто список фраз, а не инструмент. Не вызывай `wordstat_top_requests` пока не получил ответы на блок вопросов из шага 0 или пока пользователь не дал достаточно вводных в запросе.

## Режим работы

В начале определи, что именно нужно пользователю:

- `ppc_core` — семантика под Яндекс Директ: кампании, группы, ключи, минусы, гипотезы, экономика.
- `seo_core` — семантика для сайта/SEO: категории, страницы, информационные кластеры, коммерческие страницы, FAQ, смежный спрос.
- `hybrid_core` — общее ядро с разделением: что в Директ, что в SEO, что в контент, что в минус.
- `audit_or_expand` — чистка/расширение существующего ядра или поисковых запросов.

Если пользователь просит "максимально полный список ключей для сайта", "ключи для SEO", "семантику для страниц", не навязывай структуру рекламных кампаний как единственный результат. В этом режиме нужна полнота охвата спроса и группировка под страницы.

## Workflow

### Шаг 0. Брифинг (обязательно, до любых tool-calls)

Задай клиенту вопросы одним сообщением (не по одному). Если задача срочная, задай короткий минимум и явно зафиксируй допущения.

**Минимум для SEO/site-core:**
1. Ниша и что именно продаём/продвигаем.
2. Гео: страна, регион, город, язык.
3. Целевая аудитория: кто ищет и кто принимает решение.
4. Основные категории товаров/услуг: 5-10 приоритетных направлений.
5. Тип сайта: интернет-магазин, услуги, SaaS, блог, лендинг, маркетплейс.
6. Бренды, конкуренты и темы, которые нельзя использовать.

**Расширенный бриф для PPC/hybrid:**

1. **Бизнес и продукт**
   - Что продаём (категория + конкретные товары/услуги)
   - Средний чек, маржа в %, география (РФ / регион / город)
   - Сезонность (есть ли пики/спады)

2. **Юнит-экономика**
   - Целевой CPA (стоимость лида/заказа)
   - Целевой ДРР (доля рекламных расходов от выручки)
   - LTV — есть ли повторные покупки, можно ли лить в минус по первому заказу

3. **Аналитика**
   - Подключена ли Метрика, какие цели настроены (получить через `metrika_get_goals`)
   - Есть ли сквозная аналитика (Roistat, Calltouch, своя BI)
   - Какие источники сейчас работают (organic / SEO / Директ / соцсети)

4. **Конкуренты**
   - 2-3 прямых конкурента (для контекста, не для парсинга — глубокий парсинг через Keys.so/Spywords вне MCP)
   - Чем мы лучше / в чём УТП

5. **Бренд-политика**
   - Льём ли по своему бренду (защита поисковой выдачи)
   - Льём ли по конкурентам (юридические риски в РФ)
   - Есть ли запрещённые формулировки

6. **Текущая воронка**
   - Что уже работает / что не работает
   - Какие гипотезы хотим проверить
   - Бюджет на тест (минимум 30-50 кликов на гипотезу)

После ответов — сформулируй "стратегию семантики" одним абзацем и подтверди с клиентом перед объёмным сбором. Если пользователь просит не тормозить, продолжай на допущениях и пометь их в результате.

### Шаг 1. Источники

Wordstat — база, но не основа. Используй комбо:

```
wordstat_top_requests       # частотность + идеи (ассоциации правой колонки)
wordstat_dynamics           # сезонность для проверки гипотез про пики
get_search_queries          # ЕСЛИ у клиента уже есть кампании — там реальные фразы
metrika_get_popular_pages   # какие страницы УЖЕ приводят трафик (=релевантные интенты)
metrika_get_traffic_sources # источники существующего трафика
```

**Важно:** `get_search_queries` по существующим кампаниям часто ценнее Wordstat — там реальные фразы пользователей, которых нет в базе.

### Шаг 2. Чистка и кластеризация

**Не "по смыслу" — а по интенту + температуре спроса:**

| Температура | Признаки | Кампания | Ставки |
|---|---|---|---|
| Горячая (коммерческая) | купить, заказать, цена, стоимость, оптом, доставка | Отдельная, поиск | Высокие |
| Тёплая | сравнение, отзывы, лучший, рейтинг, какой выбрать | Отдельная, поиск+РСЯ | Средние |
| Холодная (инфо) | как, что такое, своими руками, инструкция | РСЯ или отсекаем | Низкие |

**По типу спроса (всегда разные кампании):**
- БРЕНДОВЫЕ — свой бренд, защита позиций, стратегия `HIGHEST_POSITION`
- КОНКУРЕНТНЫЕ — бренды конкурентов, отдельные ставки
- ОБЩИЕ — категорийные запросы, основная масса трафика

**Действия:**
1. `check_search_volume` — отсей нулевые (показов меньше порога)
2. `deduplicate_keywords` — убери дубли
3. Кросс-минусация: фразы не должны пересекаться между группами (иначе кампании конкурируют сами с собой)

### Шаг 2.5. Матрица покрытия спроса

Чтобы не собрать только очевидные ключи, проходи по матрице для каждой категории/услуги:

| Слой | Что искать | Примеры модификаторов |
|---|---|---|
| ВЧ / категория | общий спрос | категория, тип услуги, продукт |
| СЧ / характеристики | уточнения выбора | бренд, модель, материал, размер, назначение, цена, качество |
| НЧ / длинный хвост | точная потребность | для кого, для чего, проблема, действие, условия |
| Коммерческие | готовность купить | купить, заказать, цена, стоимость, доставка, рядом, под ключ |
| Вопросы | верхняя/средняя воронка | как выбрать, какой лучше, где купить, сколько стоит |
| Информационные | контент и SEO | что такое, для чего нужен, состав, инструкция, отзывы |
| Смежные | околоцелевой спрос | альтернативы, заменители, сопутствующие товары/услуги |
| Гео | локальный спрос | город, район, рядом, на дому, с доставкой |
| Бренды | брендовый спрос | свой бренд, товарные бренды, конкуренты отдельно |

Правила полноты:
- Для SEO/site-core цель — широкий охват: обычно 200-300+ уникальных запросов, если ниша не слишком узкая.
- Для PPC-core цель — не количество, а управляемые кластеры с экономикой; мусорные инфо-запросы чаще уходят в минус или SEO.
- Синонимы, разговорные варианты, транслит и частые опечатки добавляй только если они реально встречаются в Wordstat/поисковых запросах или явно типичны для ниши. Не плодить выдуманные опечатки.
- Брендовые, конкурентные и собственные навигационные запросы всегда выделяй отдельно.
- Околоцелевые запросы не смешивай с коммерческими: помечай как SEO/РСЯ/ретаргетинг/тест.

### Шаг 3. Структура под автостратегии (2024+)

- **SKAG (одна фраза = одна группа) устарел** — алгоритмам нужна статистика для обучения
- Укрупняй группы до **15-30 фраз одного интента**, но достаточно узко чтобы один текст объявления был релевантен всем фразам
- Разделение кампаний: `тип трафика (поиск/РСЯ) × гео × температура × маржинальность товара`

### Шаг 4. Длинный хвост

НЧ-запросы 1-5 показов/мес. Джуны их выбрасывают, сеньоры — заливают пакетно через `add_keywords_batch`. По отдельности копейки, в массе — дешёвый конверсионный трафик с минимальной конкуренцией.

### Шаг 5. Минус-слова — многоуровнево

| Уровень | Что туда | Как добавлять |
|---|---|---|
| Аккаунт (общий мусор) | бесплатно, скачать, своими руками, реферат, вакансии, б/у | `add_negative_keyword_set` |
| Кампания | нерелевантные категории | в `add_campaign` / `update_campaign` |
| Группа | фразы соседних групп (кросс-минусация) | в `add_adgroup` / `update_adgroup` |

**Регулярность:**
- Крупные кампании — ежедневный `get_search_queries`
- Стандарт — еженедельно
- Сразу `update_negative_keyword_set` после анализа

### Шаг 6. Связка с экономикой (главное отличие от джуна)

Через 2-4 недели после запуска:

```
get_criteria_stats        # CPA и ДРР по каждой фразе/группе
metrika_get_goals_report  # фраза → достижение цели
metrika_get_direct_report # связка Директ + Метрика
```

Алгоритм:
1. По каждому кластеру посчитай реальный CPA и ДРР
2. Кластеры с CPA выше целевого × 1.5 — отключай или снижай ставки
3. Кластеры с CPA в норме — масштабируй (расширение фраз через `wordstat_top_requests` + LAL)
4. Если есть LTV-данные — горячие коммерческие кластеры могут окупаться вторым заказом, тогда поднимай CPA-цель

### Шаг 7. Тестирование как продукт

- Не "настраиваем кампанию", а **ведём гипотезы**: "Кластер X должен дать CPA ≤ Y за 2 недели"
- Тестируй не только объявления, но и **сами кластеры** — какие группы запросов вообще стоит закупать
- Документируй: `workspace_add_tasks` с описанием гипотезы и датой проверки

## Output

Формат зависит от режима.

### Output для PPC/hybrid

После сбора отдай клиенту:

1. **Бриф-сводку** — что узнал, какая стратегия
2. **Структуру кампаний** — список кампаний с описанием цели каждой
3. **Кластеры по группам** — таблица "группа → 15-30 фраз → температура → ожидаемая ставка"
4. **Минус-слова** — три списка (аккаунт / кампании / группы)
5. **Гипотезы для теста** — 3-5 штук с метриками успеха и сроком
6. **Готовые tool-calls для запуска** — последовательность `add_campaign → sitelinks → add_adgroups → add_keywords_batch → add_ads`

### Output для SEO/site-core

Если пользователь просит семантическое ядро для сайта или полный список ключей:

1. **Сводка ядра** — ниша, гео, аудитория, источники, принятые допущения.
2. **Структура страниц** — категории/посадочные/статьи, под которые группируется спрос.
3. **Группы запросов** — по категориям, каждый запрос с новой строки или в таблице.
4. **Интент** — commercial / informational / mixed / navigational.
5. **Частотность** — если Wordstat доступен; если нет, явно пометь "частотность не проверена".
6. **Брендовые запросы** — отдельный блок: свой бренд, товарные бренды, конкуренты.
7. **Смежный спрос** — отдельный блок, не смешивать с основным commercial.
8. **Минус-слова / исключения** — что не брать в рекламу или не использовать на сайте.

Минимальный формат для запроса "просто дай список":

```text
Категория: <название>
Интент: <commercial / informational / mixed>
Посадочная: <url или тип страницы>

<ключ 1>
<ключ 2>
<ключ 3>
...
```

Если ядро большое, сначала покажи структуру категорий и 20-30 примеров, затем предложи выгрузить полный список в файл.

## Anti-patterns (как НЕ надо)

- ❌ Сразу `wordstat_top_requests "пицца москва"` без вопросов про бизнес
- ❌ Заливка всего что нашёл в Wordstat без кластеризации
- ❌ Для SEO-задачи отдавать только структуру рекламных кампаний и игнорировать страницы сайта
- ❌ Собирать 200-300 фраз механическими перестановками без проверки интента и дублей
- ❌ Одна кампания на бренд + конкурентов + общие запросы
- ❌ Группа из одной фразы (SKAG) — не даёт статистики автостратегиям
- ❌ Минус-слова "потом добавим" — добавляются ДО запуска
- ❌ Игнор длинного хвоста как "нерентабельного для проработки"
- ❌ Отчёт "собрал N фраз" — должен быть отчёт "собрал N кластеров с прогнозом CPA"

## Связи с другими инструментами

После сбора семантики типичный пайплайн:
```
add_campaign              # создание кампании
sitelinks + ad_extensions # быстрые ссылки и уточнения
add_adgroups              # группы по интентам
add_keywords_batch        # фразы пакетно
add_negative_keyword_set  # минус-слова
add_ads / add_responsive_ad # объявления (2-3 варианта на группу)
workspace_add_tasks       # напоминания о проверке гипотез через 7/14/28 дней
```
