# Scenario Router

Используй после брифа, если задача шире обычного поискового запуска.

## Базовая развилка

- **Search-first services/leads**: `TEXT_CAMPAIGN`, Поиск, фразы, минусы, 2-3 объявления на группу. Это default.
- **Search-first ecommerce**: `UNIFIED_CAMPAIGN` допустима для товарной галереи, страниц каталога, комбинаторных объявлений, Карт или списка организаций. Сети выключены, если пользователь не просил РСЯ.
- **Small budget**: не дроби кампании по каждому месту показа и типу объявления, если каждая часть не наберёт данные. Укрупнение допустимо ради обучения, но теряется управление долями трафика.
- **Large budget / mature account**: разделяй по местам показа, типам объявлений, категориям, регионам и стратегиям, если это нужно для бюджета, отчётности и управления эффективностью.

## ЕПК

ЕПК нужна, когда есть причина использовать возможности, которых нет в обычной текстовой кампании: товарная галерея, товарные объявления, страницы каталога, комбинаторные объявления, Карты, список организаций, маркетплейс с интеграцией.

Не используй ЕПК как скрытый Мастер кампаний. Через API это должен быть управляемый сценарий: `add_unified_campaign`, затем группы, элементы, объявления, модерация только после отдельного подтверждения.

## РСЯ, ретаргетинг, look-alike

РСЯ и аудиторные сценарии работают не как Поиск:

- в Поиске ключи и автотаргетинг отвечают на запрос;
- в РСЯ таргетинги могут расширять аудиторию через интересы, привычки, площадки и пользователей;
- несколько аудиторных условий в РСЯ часто работают шире, чем ожидает клиент.

Перед РСЯ/ретаргетингом/LAL покажи search-first план, объясни риск качества трафика, запроси текстовое подтверждение и создай отдельные monitoring tasks.

## Маркетплейсы

- Яндекс Маркет и Ozon с интеграцией продаж: можно рассматривать ЕПК с оптимизацией на покупки.
- Wildberries, Авито и Ozon без интеграции: не обещай оптимизацию на покупки через ЕПК. Если пользователь просит такой запуск, объясни ограничение и предложи отдельный сценарий с доступными прокси-целями.

## Медийка и приложения

Медийка — верх воронки, CPM/CPV, охват, частота, lift/post-view. Приложения — AppMetrica, установки, события, трекер, CPI/CPA. Не смешивай их с обычной search-first кампанией без отдельного плана аналитики.
