# Semantic Core — семантическое ядро под экономику, SEO и кампании

Используй этот skill, когда пользователь просит собрать, расширить, почистить, кластеризовать или подготовить семантическое ядро для Яндекс Директа, SEO или лендинга.

Главный принцип: семантика не является списком ключей. Это структура спроса под экономику, интент, посадочные страницы, SEO-архитектуру и будущую оптимизацию.

## Быстрый Старт

1. Определи режим работы:
- `new_core` — ядро с нуля.
- `seo_site_core` — ядро для сайта/SEO: категории, посадочные, статьи, FAQ, смежный спрос.
- `hybrid_core` — общее ядро с разделением: что в Директ, что в SEO, что в контент, что в минус.
- `expand_core` — расширение существующей семантики.
- `clean_cluster` — чистка, кластеризация, кросс-минусация.
- `audit_core` — аудит текущих ключей и поисковых запросов.
- `campaign_structure` — перевод ядра в структуру кампаний/групп.

Если пользователь просит "ключи для сайта", "SEO ядро", "максимально полный список запросов", "группируй по категориям", выбирай `seo_site_core`, а не только рекламную структуру.

2. Если не хватает вводных, задай короткие вопросы. Для списка вопросов читай `references/intake-questions.md`.

3. Сначала выясни экономику:
- продукт/услуга и маржинальность;
- целевое действие;
- допустимый CPA или ДРР;
- гео;
- посадочные страницы;
- ограничения по спросу, брендам, конкурентам, юридике.

Для `seo_site_core` экономика важна, но минимум вводных другой: ниша, гео, аудитория, категории, тип сайта, бренды/конкуренты и запретные темы.

4. Используй источники в порядке приоритета:
- текущие данные бизнеса: сайт, посадочные, CRM/сквозная аналитика, если есть;
- поисковые запросы Директа и Метрика, если есть активные кампании;
- Wordstat: базовые фразы, точная частотность, ассоциации, динамика, регионы;
- конкуренты и SERP, если нужна проверка интента или расширение;
- внешние базы/выгрузки, если пользователь их дал.

5. Результат всегда отдавай не как сырой список, а как управляемую структуру: кампании, группы/кластеры, ключи, минусы, температура спроса, посадочная, приоритет, риск и следующий тест.

Для SEO/site-core результат отдавай как структуру страниц и контентных кластеров: категории, коммерческие посадочные, информационные статьи, FAQ, брендовые и смежные блоки. Если пользователь просит именно список, группируй запросы по категориям, но всё равно показывай интент и назначение страницы.

## MCP Инструменты

Для Wordstat используй MCP-сервер `yandex-direct`:

| Инструмент | Когда использовать |
|---|---|
| `wordstat_user_info` | Перед объёмным сбором, чтобы проверить квоты |
| `wordstat_top_requests` | Базовая частотность, расширение фраз, ассоциации |
| `wordstat_dynamics` | Сезонность и бюджетные ожидания |
| `wordstat_regions` | Региональный спрос и приоритизация гео |
| `wordstat_regions_tree` | Справочник регионов; формат узла `{ value, label, children }` |

Для действующих кампаний используй `yandex-direct-campaign-builder` и `yandex-metrika`:

| Данные | Инструменты |
|---|---|
| Текущие кампании и группы | `get_campaigns`, `get_adgroups`, `get_keywords` |
| Реальные поисковые запросы | `get_search_queries` |
| Расход, клики, конверсии | `get_campaign_stats`, `get_adgroup_stats`, `get_criteria_stats` |
| Поведение и цели на сайте | `metrika_*`, отчёты по UTM и целям |

Если нужно применить изменения в Директе, сначала покажи план изменений и дождись явного текстового подтверждения пользователя.

## Workflow: Новое Ядро

1. Собери вводные.
- Не начинай массовый сбор, пока не понятны продукт, гео, целевое действие, посадочные и экономика.
- Если данных нет, зафиксируй рабочие допущения отдельно.

2. Составь seed list.
- Названия продукта/услуги.
- Категории, синонимы, боли, jobs-to-be-done.
- Коммерческие модификаторы: купить, заказать, цена, стоимость, под ключ, рядом, доставка.
- SEO-модификаторы: как выбрать, какой лучше, где купить, сколько стоит, что такое, для чего нужен, отзывы, сравнение, инструкция.
- Брендовые и конкурентные запросы отдельными списками.

3. Расширь через Wordstat.
- Проверь квоты через `wordstat_user_info`.
- Для каждой seed-фразы вызови `wordstat_top_requests`.
- Для сезонных ниш добавь `wordstat_dynamics`.
- Для региональной рекламы добавь `wordstat_regions`.

4. Обогати first-party данными.
- Если есть активные кампании, выгрузи `get_search_queries` минимум за 30 дней, лучше 90 дней.
- Сравни реальные запросы с текущими ключами через `get_keywords`.
- Сохрани конверсионные запросы как кандидаты на отдельные ключи/кластеры.

5. Классифицируй интент и температуру.
- Читай `references/intent-classification.md`.
- Для SEO/site-core также читай `references/seo-site-core.md`.
- Разделяй горячие, тёплые, холодные, брендовые, конкурентные, общие.
- Информационные запросы не удаляй автоматически: для поиска часто в минус, для SEO/контента можно вынести отдельно.

6. Кластеризуй.
- Читай `references/clustering-rules.md`.
- Под автостратегии не делай SKAG по умолчанию.
- Группа должна быть достаточно широкой для обучения, но достаточно узкой для релевантного объявления и посадочной.

7. Собери минус-слова.
- Читай `references/minus-words.md`.
- Делай уровни: аккаунт, кампания, группа.
- Кросс-минусацию делай только после финальной структуры кластеров.

8. Выдай результат.
- Читай `references/output-format.md`.
- Покажи: структуру кампаний, кластеры, ключи, минусы, посадочные, приоритеты, гипотезы, что проверить после запуска.
- Для SEO/site-core покажи: структуру страниц, категории, запросы по одному на строку или таблицей, интент, частотность если есть, брендовые и смежные блоки.

## Workflow: Аудит Или Расширение

1. Получи текущие кампании, группы, ключи и поисковые запросы.
2. Посчитай по кластерам: расход, клики, конверсии, CPA/ДРР, CTR, долю мусорных запросов.
3. Найди:
- конверсионные запросы, которых нет в ключах;
- запросы с расходом без конверсий;
- пересечения и каннибализацию между группами/кампаниями;
- кластеры с неподходящей посадочной;
- слишком мелкие группы, которым не хватает данных для автостратегий;
- слишком широкие группы, где теряется релевантность.
4. Предложи действия:
- добавить ключи;
- вынести кластер;
- объединить группы;
- добавить минусы;
- заменить посадочную;
- отключить или ограничить кластер по экономике.

## Senior Правила

- Не оптимизируй под максимум ключей. Оптимизируй под прибыльный спрос.
- Для SEO/site-core цель — широкий охват спроса, но не механические перестановки. Обычно 200-300+ уникальных запросов уместны только если ниша достаточно широкая и интент проверен.
- Не смешивай бренд, конкурентов и общие запросы в одной кампании.
- Не смешивай поиск и РСЯ в одной логике семантики.
- Не смешивай горячий коммерческий спрос и холодный информационный спрос без отдельной стратегии.
- Не называй кластер готовым, если у него нет посадочной страницы или ясного следующего действия.
- Не используй точную частотность как единственный критерий. НЧ и микро-НЧ могут быть ценными при хорошей автоматизации.
- Не выдумывай опечатки ради объёма: добавляй только проверенные через Wordstat/поисковые запросы или явно типичные для ниши.
- Не применяй изменения в рекламном аккаунте без явного текстового подтверждения.
